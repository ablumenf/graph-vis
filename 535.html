To choose your own graph, simply open the console up in your browser and type in the number of vertices and your adjacency lists. For example, to draw $K_5$, type the following, then click Draw Graph:

G1.numVertices = 5;
G1.adjacencyLists = [[1, 2, 3, 4], [0, 2, 3, 4], [0, 1, 3, 4], [0, 1, 2, 4], [0, 1, 2, 3]];

To draw $K_{3, 3}$, type the following, then click Draw Graph. (There are probably better algorithms out there for drawing bipartite graphs.)

G1.numVertices = 6;
G1.adjacencyLists = [[3, 4, 5], [3, 4, 5], [3, 4, 5], [0, 1, 2], [0, 1, 2], [0, 1, 2]];

You should use G1 as your variable! The way to read this is that node $0$ is connected to nodes $1$ through $4$, node $1$ is connected to node $0$ and nodes $2$ through $4$, etc.

You can also specify your node labeling. The default labels are $0$ through $n-1$, where $n$ is the number of vertices. You can change this to, say, $A, \ldots, J$ (for the default graph, which has 10 vertices) (before clicking Draw Graph) by typing in the following: 

labels1 = ["A", "B", "C", "D", "E", "F", "G", "H", "I", "J"];

Or even something more creative like (although it's probably best not to have super long labels, in interest of readability):

labels1 = ["hello", "arkona", "apkoha", "vert", "hi", "print", "what", ":)", "good", "bye"];

You can't use LaTeX code in the labels, though. Even though MathJax is used on this page, it doesn't play nice with drawing text in d3. It's probably best to create a figure using TikZ if using more complicated labels is necessary (like $\alpha_{\beta_i}$ or something). Note that the result will not be unique. The algorithm used (Fruchterman-Reingold) searches for a local minimum, and is quite unlikely to find an absolute minimum. In fact, you can repeatedly click Draw Graph and watch the graph dance around.
